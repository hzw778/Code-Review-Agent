启动 Agent 后会做什么？

Agent 采用 ReAct 模式（Reasoning + Acting），循环执行以下步骤直到完成：

  1. GitDiffTool：克隆仓库并获取该 commit 的代码变更（diff）
  2. AstAnalysisTool：对变更的 Java 文件做 AST 静态分析（空 catch、资源泄漏、方法过长等）
  3. RagSearchTool：检索 Java 规范知识库，找到与问题相关的规范条目
  4. Finish：LLM 综合以上信息，输出带规范引用的审查意见
  5. Reflection：LLM 自检，过滤误报和幻觉，输出最终报告

每一步的思考、工具参数、返回结果都会实时显示在右侧"数据传输轨迹"面板。
